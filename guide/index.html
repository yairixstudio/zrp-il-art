QR Campaigns

Turn any printed ad, window display or in-store card into a measurable sign-up. Each QR code carries the store, city and country it was printed for, so every new subscriber is tagged with where they came from.

How it works

  1. Generate a code. Fill in the store, city, country and any extra details in the generator. A QR code and link are built live.
  2. Print or place it. Download the QR as PNG (raster) or SVG (vector, best for print) and add it to the artwork.
  3. Customer scans. The code opens the sign-up form, already aware of its location and campaign.
  4. Data lands in Klaviyo. The email and all the tagged details are sent to your list as profile properties.

Parameters

Everything you add in the generator is appended to the form link as query parameters and forwarded to Klaviyo as profile properties.

ParameterMeaning
store_idIdentifier of the physical store or point of sale.
cityCity the ad is placed in.
countryTwo-letter country code; the form expands it to the full name.
langLanguage the form opens in. Defaults to English when omitted.
campaignFree-text reference for the ad or placement.
utm_source, utm_medium, utm_campaignStandard attribution tags, added automatically.
customAny extra key/value pairs you add in the generator.

Example link

/form/?store_id=0042&city=Paris&country=FR&lang=fr&campaign=spring-window

Embedding the form

Besides a QR code, the form can be embedded directly in any web page or landing page with an <iframe>. Adjust the parameters per placement.

<iframe
  src="HOST/form/?store_id=0042&city=Paris&country=FR"
  width="100%" height="640" style="border:0"
  title="Subscribe" loading="lazy"></iframe>

Languages

Connecting Klaviyo

Open form/index.html and set the two values at the top of its script. companyId is the public API key (site id); listId is the target list.

var KLAVIYO = { companyId: "XXXXXX", listId: "YYYYYY" };
Until both values are filled in, the form runs in preview mode: it shows the thank-you screen but does not send anything.
